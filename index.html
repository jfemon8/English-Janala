Frequently Asked Questions

What is the difference between var, let, and const?
var is function-scoped, which means its value is accessible throughout the function where it is declared. It can be re-declared and updated. let is block-scoped, meaning it is only accessible within the block ({}) where it's declared. It can be updated but cannot be re-declared in the same scope. const is also block-scoped like let but is used to declare constants. Once assigned, its value cannot be changed or re-declared in the same scope.
What is the difference between map(), forEach(), and filter()?
map() creates a new array by transforming each element of the original array based on a provided function. It returns a new array with the same length. forEach() executes a function for each element in the array but does not return anything. It's mainly used for performing side effects like logging, updating UI, or modifying external variables. filter() creates a new array by including only those elements that pass a specific condition. It returns a new array with elements that meet the condition.
Explain arrow functions and how they are different from regular functions.
Arrow functions provide a shorter syntax for writing functions. The biggest difference is how they handle the this keyword. In arrow functions, this refers to its surrounding scope, meaning it inherits this from its parent function or context. Regular functions have their own this depending on how they are called.
How JavaScript Promises work?
Promises in JavaScript handle asynchronous operations, like fetching data from an API or reading files. A promise has three states: Pending, Fulfilled and Rejected. Promises provide .then() for handling success and .catch() for handling errors.
How closures work in JavaScript?
A closure happens when a function remembers and continues to access variables from its outer function even after that parent function has finished executing. In simple terms, closures allow inner functions to access variables from an outer function's scope, preserving data in memory even after the outer function is done running.
